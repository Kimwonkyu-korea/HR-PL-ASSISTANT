# HR PL Assistant — 화면 설계서 (와이어프레임)

**작성일:** 2026-03-31
**버전:** v0.1 MVP
**대상 화면:** SCR-001 ~ SCR-005

---

## 공통 레이아웃 원칙

- **해상도 기준:** 1440px (데스크탑 우선)
- **레이아웃 구조:** 좌측 사이드바(240px) + 우측 콘텐츠 영역
- **Split View:** 생성 화면은 좌(입력) / 우(결과) 50:50 분할
- **색상 톤:** 화이트 베이스, 포인트 컬러 Indigo-600
- **폰트:** Pretendard (한국어 최적화)

---

## 공통 컴포넌트

### GNB (Global Navigation Bar) — 상단 고정

```
┌─────────────────────────────────────────────────────────────────────┐
│  ◈ HR PL Assistant          [프로젝트: 미설정 ▾]        [⚙ 설정]    │
└─────────────────────────────────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 로고 | 클릭 시 홈(/)으로 이동 |
| 프로젝트 드롭다운 | 현재 세션의 프로젝트명 표시. 클릭 시 설정 페이지로 이동 |
| 설정 아이콘 | SCR-005 설정 페이지로 이동 |

---

### 사이드바 (LNB) — 좌측 고정

```
┌────────────────┐
│ 문서 생성       │
│ ─────────────  │
│ 📋 회의록      │  ← active 시 Indigo 배경
│ 📄 요구사항     │
│ 🖥 화면정의서   │  ← [MVP 이후] 비활성 처리
│ 🔄 마이그레이션 │  ← [MVP 이후] 비활성 처리
│ ✅ 테스트케이스 │
│                │
│ 흐름 연결       │
│ ─────────────  │
│ ◈ 현재 세션     │
│   회의록 ✓     │  ← 완료 문서 체크 표시
│   요구사항 ✓   │
│   테스트 –     │
└────────────────┘
```

| 요소 | 설명 |
|------|------|
| 메뉴 항목 | 각 문서 생성 페이지로 이동 |
| MVP 이후 항목 | 비활성(disabled) 스타일, 툴팁: "v1.0에서 제공 예정" |
| 세션 흐름 패널 | 현재 세션에서 완료된 문서 단계 표시. 완료 시 ✓, 미완료 시 – |

```
│            │  │ 회의 정보            │                              ││
│            │  │ 프로젝트명: [_____] │  (생성 전 상태)               ││

│            │  │ │  - 레거시DB     │ │  │ ## 주요 논의사항         │ ││

└────────────┴────────────────────────────────────────────────────────┘
```

### 컴포넌트 상세

| 컴포넌트 | 타입 | 동작 |
|----------|------|------|
| 프로젝트명 | Text Input | 설정에서 값 있으면 자동 채움 |
| 회의일시 | Date-Time Input | 기본값: 오늘 날짜 |
| 회의장소 | Text Input | 자유 입력 |
| 참석자 | Tag Input | 쉼표 또는 Enter로 태그 추가 |
| 회의 메모 | Textarea (resizable) | 최소 높이 200px, 자유 텍스트 |
| 추가 지시사항 | Text Input | AI 프롬프트에 추가 전달될 보조 지시 |
| [✨ 회의록 생성] | Primary Button | 클릭 시 API 호출, 로딩 스피너 표시 |
| 결과 영역 | Markdown Renderer | 생성된 마크다운 렌더링 |
| [📝 편집] | Toggle Button | 클릭 시 결과 영역을 편집 가능한 Textarea로 전환 |
| [📥 Word 다운로드] | Button | docx.js로 클라이언트 사이드 생성 후 다운로드 |
| [→ 요구사항으로] | CTA Button | 현재 결과를 sessionStorage에 저장 후 SCR-003으로 이동 |

### 상태 흐름

```
[초기] → 입력 → [생성 중: 로딩] → [결과 표시] → 편집 가능
                                              ↓
                                    다운로드 or 다음 단계
```

### AI 프롬프트 구조 (참고)

- **System:** HR 시스템 구축 프로젝트 전문 회의록 작성자 역할
- **User:** 회의 정보 + 메모 + 추가 지시사항
- **출력 형식:** 마크다운 (참석자 / 주요 논의사항 / 결정사항 / 액션아이템 테이블)


### 컴포넌트 상세

| 컴포넌트 | 타입 | 동작 |
|----------|------|------|
| 요구사항 연결 배너 | Info Banner | sessionStorage에 요구사항 있을 때 표시 |

### 테스트 케이스 결과 테이블 컬럼 정의

| 컬럼 | 설명 |
|------|------|
| TC-ID | 고유 식별자 (TC-001 형식) |
| 요구사항 ID | 연결된 요구사항 (F-01 등) |
| 테스트 항목 | 테스트 내용 요약 |
| 테스트 유형 | 정상/예외/경계값 |

## 인터랙션 흐름 정의

### 핵심 흐름: 회의록 → 요구사항 → 테스트케이스

```
SCR-002 회의록 생성
    ↓ [→ 요구사항 정의서로 이어서] 클릭
    • 회의록 결과를 sessionStorage['minutes'] 에 저장
    • SCR-003으로 navigate

SCR-003 요구사항 정의서 생성
    • sessionStorage['minutes'] 존재 → 연결 배너 표시
    • [✨ 요구사항 생성] 클릭 → 회의록 컨텍스트 포함하여 API 호출
    ↓ [→ 테스트 케이스로 이어서] 클릭
    • 요구사항 결과를 sessionStorage['requirements'] 에 저장
    • SCR-004로 navigate

SCR-004 테스트 케이스 생성
    • sessionStorage['requirements'] 존재 → 연결 배너 표시
    • 요구사항의 기능 목록 파싱 → 체크박스 자동 생성
    • 선택된 항목 기반으로 API 호출
```

### 에러 처리 패턴

| 상황 | 처리 방식 |
|------|-----------|
| API Key 미설정 | 생성 버튼 클릭 시 모달: "설정에서 API Key를 입력해주세요" |
| API 호출 실패 | 결과 영역에 에러 메시지 + [다시 시도] 버튼 |
| 입력값 없음 | 생성 버튼 비활성화 (disabled) |
| 생성 중 이탈 | 브라우저 beforeunload 경고 |
