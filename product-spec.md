# HR PL Assistant — 제품 기획서

**작성일:** 2026-03-31
**버전:** v0.1 (MVP)
**대상 사용자:** HR 시스템 구축 PM / PL

---

## 1. 제품 개요

### 1.1 배경 및 목적

HR 시스템 구축 프로젝트에서 PM/PL은 반복적으로 동일한 유형의 문서를 작성한다.
회의 메모 → 요구사항 정의서 → 화면 정의서 → 마이그레이션 설계서 → 테스트 케이스로 이어지는 문서 체계는 논리적 흐름이 있음에도, 매번 수작업으로 작성되고 있다.

**HR PL Assistant**는 이 흐름을 AI로 자동화하여 문서 작성 시간을 단축하고 품질을 표준화하는 웹 기반 도구다.

### 1.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **속도** | 회의 메모 입력 → 30초 내 초안 생성 |
| **일관성** | HR 도메인 특화 템플릿으로 문서 품질 표준화 |
| **독립성** | DB 없음, 로그인 없음, 브라우저에서 즉시 사용 |
| **연결성** | 문서 간 컨텍스트 전달로 흐름 기반 자동화 |

### 1.3 사용하지 않는 것 (Non-Goals)

- 사용자 계정 관리 / 로그인
- 서버 사이드 DB 저장
- 팀 협업 / 실시간 공유
- 승인 워크플로우

## 2. 사용자 및 시나리오

### 2.1 대상 사용자

- **주 사용자:** HR 시스템 구축 프로젝트의 PM / PL
- **경력:** HR IT 프로젝트 경험 3년 이상, 문서 작성 부담을 느끼는 실무자
- **환경:** 사내 PC 또는 노트북, Chrome/Edge 브라우저

### 2.2 핵심 사용 시나리오

```
[현장 회의]
    ↓ 메모 입력 (10분)
[회의록 자동 생성]
    ↓ 주요 결정사항 추출
[요구사항 정의서 자동 생성]
    ↓ 요구사항 선택/편집
[화면 정의서 초안 생성]
    ↓ 화면 목록 기반
[마이그레이션 설계서 생성]
    ↓ 영향 범위 분석
[테스트 케이스 생성]
    ↓
[Word/Excel 다운로드]
```

## 3. 기능 정의

### 3.1 핵심 기능 (Core Features)

| # | 기능명 | 입력 | 출력 | 비고 |
|---|--------|------|------|------|
| F-01 | 회의록 생성 | 회의 메모(자유 텍스트), 회의 정보 | 구조화된 회의록 | 참석자, 결정사항, 액션아이템 추출 |
| F-02 | 요구사항 정의서 생성 | 회의록 or 직접 입력 | 요구사항 정의서 | 기능/비기능 요구사항 분류 |
| F-03 | 화면 정의서 생성 | 요구사항 정의서 or 직접 입력 | 화면 목록 + 각 화면 정의 | 화면ID, 컴포넌트, 이벤트 정의 |
| F-04 | 마이그레이션 설계서 생성 | 요구사항 + 화면 정의 or 직접 입력 | 마이그레이션 설계서 | 데이터 매핑, 전환 전략 |
| F-05 | 테스트 케이스 생성 | 요구사항 or 화면 정의 | 테스트 케이스 목록 | 정상/예외/경계값 케이스 포함 |

### 3.2 공통 기능

| # | 기능명 | 설명 |
|---|--------|------|
| C-01 | 문서 편집 | 생성된 문서를 브라우저 내에서 인라인 편집 |
| C-02 | Word 다운로드 | .docx 형식으로 다운로드 |
| C-03 | Excel 다운로드 | .xlsx 형식으로 다운로드 (테이블 중심 문서) |
| C-04 | 컨텍스트 전달 | 이전 단계 문서를 다음 단계 입력으로 자동 연결 |
| C-05 | 세션 임시 저장 | 브라우저 sessionStorage에 작업 내용 임시 보관 |
| C-06 | 프롬프트 커스터마이징 | 프로젝트명, 조직명 등 메타정보 설정 |

## 4. MVP 범위

### 4.1 MVP 포함 항목

| 우선순위 | 기능 | 이유 |
|----------|------|------|
| P0 | F-01 회의록 생성 | 파이프라인 시작점, 가장 자주 사용 |
| P0 | F-02 요구사항 정의서 생성 | 회의록 → 요구사항 핵심 흐름 |
| P0 | C-02 Word 다운로드 | 최소 결과물 전달 수단 |
| P1 | F-05 테스트 케이스 생성 | 요구사항 기반 직접 생성 가능, 수요 높음 |
| P1 | C-01 문서 편집 | 생성 후 수정 없으면 실용성 저하 |
| P1 | C-04 컨텍스트 전달 | 흐름 자동화의 핵심 |
| P2 | F-03 화면 정의서 생성 | MVP 이후 추가 |
| P2 | F-04 마이그레이션 설계서 생성 | MVP 이후 추가 |
| P2 | C-03 Excel 다운로드 | MVP 이후 추가 |

### 4.2 MVP 성공 기준

- 회의 메모 입력 후 요구사항 정의서까지 5분 이내 생성
- 생성된 문서를 Word로 다운로드 가능
- 브라우저 새로고침 없이 회의록 → 요구사항 흐름 완주 가능

## 5. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| Frontend | Next.js 14 (App Router) | SSR 불필요하나 생태계/DX 우수 |
| UI | Tailwind CSS + shadcn/ui | 빠른 프로토타이핑 |
| AI | Claude API (claude-sonnet-4-6) | HR 도메인 한국어 품질 우수 |
| 문서 생성 | docx.js | 브라우저 내 Word 생성 |
| 상태 관리 | Zustand | 경량, sessionStorage 연동 용이 |
| 저장소 | sessionStorage only | DB 없는 아키텍처 |

## 6. 정보 아키텍처 (IA)

```
홈 (/)
├── 회의록 생성 (/minutes)
│   ├── 입력 폼
│   ├── AI 생성 결과
│   └── 편집 + 다운로드
├── 요구사항 정의서 (/requirements)
│   ├── 입력 (회의록 연결 or 직접)
│   ├── AI 생성 결과
│   └── 편집 + 다운로드
├── 화면 정의서 (/screen-spec) [MVP 이후]
├── 마이그레이션 설계서 (/migration) [MVP 이후]
├── 테스트 케이스 (/test-cases)
│   ├── 입력 (요구사항 연결 or 직접)
│   ├── AI 생성 결과
│   └── 편집 + 다운로드
└── 설정 (/settings)
    └── 프로젝트 메타정보 설정
```

## 7. 화면 설계 방향

### 7.1 레이아웃 원칙

- **Split View**: 좌측 입력, 우측 생성 결과 (데스크탑 기준)
- **단계 표시 바**: 상단에 `회의록 → 요구사항 → 화면정의 → 마이그레이션 → 테스트` 흐름 시각화
- **컨텍스트 배지**: 이전 단계 문서가 연결되어 있을 경우 배지로 표시

### 7.2 주요 화면 목록

| 화면 ID | 화면명 | 설명 |
|---------|--------|------|
| SCR-001 | 홈 / 대시보드 | 5개 도구 카드, 최근 세션 문서 목록 |
| SCR-002 | 회의록 생성 | 회의정보 + 메모 입력, 결과 편집 |
| SCR-003 | 요구사항 정의서 생성 | 이전 문서 연결, 결과 테이블 편집 |
| SCR-004 | 테스트 케이스 생성 | 요구사항 선택, 케이스 테이블 생성 |
| SCR-005 | 설정 | 프로젝트명, 고객사, 담당자 등 메타정보 |
